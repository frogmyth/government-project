# AI 흔적 제거 및 인간화 리뷰 스킬

AI가 생성한 문서를 사람이 직접 작성한 것처럼 자연스럽게 교정합니다.
정부 R&D 지원사업 신청서, 기술 제안서 등 공식 문서에 특화되어 있습니다.

---

## 사용 시점

- 정부지원사업 계획서 최종 검토 시
- AI로 초안 작성 후 제출 전 교정 시
- 외부 심사위원 제출 전 AI 흔적 점검 시
- 사업계획서, 기술개요서, 연구개발계획서 작성 완료 후

---

## 적용 규칙

이 스킬은 `humanization-rules.md` 파일의 규칙을 따릅니다.

### 핵심 원칙

1. **이모지 제거**: 정부 공식문서에 이모지는 부적절
2. **과장 수식어 완화**: "세계 최초/유일" → 근거가 있는 경우만 1회 사용
3. **문장 길이 불균일화**: 모든 문장이 같은 길이면 AI 의심 → 길고 짧은 문장 섞기
4. **접속사 다양화**: "또한, 또한, 또한" 반복 → "한편, 이에 더해, 아울러" 등 교체
5. **수동태/능동태 혼용**: AI는 능동태 일변도 → 적절히 수동태 섞기
6. **불완전한 정보 허용**: AI는 빈틈없이 채우려 함 → 일부 "추후 확인 필요" 허용
7. **코드 블록 제거**: 정부 제안서에 코드 불필요 → 자연어 설명으로 대체
8. **체크마크 정리**: ✅/❌ → "지원함/미지원" 또는 "O/X"
9. **반복 구문 제거**: 같은 내용이 여러 섹션에 반복되면 통합
10. **톤 불균일화**: 전체적으로 균일한 톤 → 섹션별로 미세하게 다르게

## 수행 절차

### 1단계: AI 흔적 탐지

대상 문서를 읽고 아래 항목을 점검합니다:

| 점검 항목 | AI 의심 패턴 | 인간적 대안 |
|-----------|-------------|-----------|
| 이모지 | 제목/소제목에 이모지 사용 | 이모지 전부 제거 |
| 수식어 | "세계 최초/유일/혁신적" 3회 이상 | 1회만 유지, 나머지 완화 |
| 문장 구조 | 모든 bullet이 동일 패턴 | 일부를 서술형으로 변환 |
| 접속사 | 같은 접속사 연속 사용 | 다양한 접속사로 교체 |
| 나열 패턴 | 3개씩 완벽한 병렬 나열 | 2개 또는 4개로 변형 |
| 코드 블록 | 제안서 본문에 코드 포함 | 자연어 설명으로 대체 |
| 볼드 남용 | 핵심어마다 **볼드** | 정말 중요한 것만 볼드 |
| 완벽한 표 | 빈 셀 없이 완벽하게 채워진 표 | 일부 셀에 "-" 또는 "확인 중" |

### 2단계: 교정 적용

규칙에 따라 문서를 수정합니다. 내용의 정확성은 유지하되, 표현과 구조를 인간적으로 변환합니다.

### 3단계: 최종 검토

교정된 문서를 다시 읽고 "이 문서가 AI가 쓴 것 같은가?"를 자가 점검합니다.

---

## 사용 예시

### 문서 검토 요청
```
사용자: 이 계획서 AI 흔적 검토해줘
AI: [문서 분석 → AI 패턴 탐지 → 교정 제안]
```

### 자동 교정 요청
```
사용자: 작성완료 폴더 문서들 인간화 처리해줘
AI: [모든 문서 순회 → humanization-rules 적용 → 수정 완료]
```

### 특정 섹션 교정
```
사용자: 1-2절 경쟁사 비교 부분 AI 티 나는 거 고쳐줘
AI: [해당 섹션만 교정]
```

---

## 주의사항

- **내용 정확성은 절대 변경하지 않음**: 수치, 날짜, 고유명사, 기술 사양은 그대로 유지
- **서식 요구사항 준수**: hwpx 서식에 복사할 내용은 서식 구조 유지
- **과도한 교정 금지**: 너무 어색하게 만들지 말 것. 자연스러움이 목표
- **정부 문서 톤 유지**: 격식체(합니다/습니다) 기본, 간결하고 명확한 표현
