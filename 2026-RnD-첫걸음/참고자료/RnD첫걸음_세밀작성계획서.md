### 1.2 디딤돌 계획서의 실제 부족 내용 분석

#### [T1] 기존 기술 대비 비교분석
- **디딤돌에서 쓴 내용**: "기존 플레이어: 단순 H/W 제조사나 범용 CMS S/W 기업이 대다수" (1줄)
- **문제**: 구체적 경쟁사 이름, 제품명, 기능 비교표 없음. 자사 기술과의 정량적 비교 부재
- **심사위원 기대**: 국내외 경쟁 기술 5개 이상의 실명 비교표 + 각 기술의 한계점 + 자사 차별화 정량 수치

#### [T2] 개발 방법의 원리/절차/방법
- **디딤돌에서 쓴 내용**: 5-Step AI Workflow를 서술했으나 각 단계가 "What"(무엇을) 중심이고 "How"(어떻게) 부족
- **문제**: sLLM 파인튜닝 → LoRA 기법 언급만 하고 수학적 원리, 학습 파라미터, 데이터 전처리 절차 미기술
- **심사위원 기대**: 각 핵심 기술마다 (1)기술 원리 (2)구현 절차(플로우차트) (3)구체적 방법(프레임워크/모델/파라미터)

#### [B1] 목표 시장 분석
- **디딤돌에서 쓴 내용**: "글로벌 RMN 시장 2,000억 달러, 연평균 15% 성장"
- **문제**: TAM/SAM/SOM 분리 없음. 국내 시장 규모 수치 없음. 시장 세그먼트 분석 없음
- **심사위원 기대**: TAM→SAM→SOM 3단계 시장 규모(출처 포함) + 세그먼트별 성장률 + 진입 가능 시장 특정

#### [B2] 판매처/시장 진입 전략
- **디딤돌에서 쓴 내용**: "대형 유통사/프랜차이즈(Primary)", "중소형 마트/소상공인(Secondary)"
- **문제**: 구체적 판매처 실명 없음. LOI/MOU 등 수요 확보 증빙 없음

#### [B3] 매출 산정 근거
- **디딤돌에서 쓴 내용**: "2028년 30억, 2029년 80억, 2030년 150억"
- **문제**: Bottom-up 산출식 없음. "200개 매장 확보"의 근거 없음

#### [B4] 경쟁사 분석
- **디딤돌에서 쓴 내용**: 경쟁사 이름 3개 (VusionGroup, Pricer) + 일반적 우위 서술

### 3.2 [T2 보완] 핵심 기술 개발의 원리/절차/방법 → 3단계 상세화

#### 각 핵심 기술별 서술 구조 (반드시 준수)

```
[기술 X] 제목
├── 1) 기술 원리 (WHY & WHAT)
│   ├── 이론적 배경 (학술 논문/표준 인용)
│   ├── 적용 원리 설명 (수식/개념도)
│   └── 선행 연구와의 관계
│
├── 2) 구현 절차 (HOW - Process)
│   ├── Step-by-step 플로우차트
│   ├── 입력 → 처리 → 출력 데이터 흐름
│   └── 각 단계별 기술적 판단 기준
│
└── 3) 구체적 방법 (DETAIL)
    ├── 사용 프레임워크/라이브러리/모델
    ├── 핵심 파라미터 (하이퍼파라미터, 임계값 등)
    ├── 학습 데이터셋 규모 및 구성
    └── 예상 소요 자원 (GPU, 시간, 비용)
```

#### 핵심 기술 1: 리테일 특화 sLLM 파인튜닝

**1) 기술 원리**
- **이론적 배경**: 대규모 언어모델(LLM)의 도메인 적응을 위한 Parameter-Efficient Fine-Tuning (PEFT) 기법 중 LoRA (Low-Rank Adaptation, Hu et al., 2021, arXiv:2106.09685)
- **적용 원리**: 사전학습된 모델의 가중치 행렬 W에 저차원 행렬 분해(rank decomposition) ΔW = BA를 추가하여, 전체 파라미터의 0.1~1%만 학습하면서도 풀 파인튜닝에 근접한 성능 달성
- **수식**: h = Wx + ΔWx = Wx + BAx (B ∈ R^{d×r}, A ∈ R^{r×k}, r << min(d,k))

**2) 구현 절차**
```
[데이터 수집] → [데이터 전처리/클리닝] → [데이터셋 포맷팅(Instruction-Response)]
→ [베이스 모델 로딩(Llama 3 8B)] → [LoRA 어댑터 설정(rank=16, alpha=32)]
→ [학습 실행(4-bit QLoRA)] → [검증(Validation Loss 모니터링)]
→ [모델 병합(Merge)] → [추론 최적화(vLLM/TensorRT)] → [API 서빙(FastAPI)]
```

**3) 구체적 방법**
| 항목 | 상세 |
|------|------|
| 베이스 모델 | Meta Llama 3 8B-Instruct (HuggingFace) |
| 파인튜닝 기법 | QLoRA (4-bit quantization + LoRA) |
| LoRA 파라미터 | rank=16, alpha=32, dropout=0.05 |
| 학습률 | 2e-5 (cosine scheduler, warmup 10%) |
| 에폭 | 5 epochs |
| 배치 사이즈 | 4 (gradient accumulation 4 = effective 16) |
| 학습 데이터 | 리테일 마케팅 문구 50,000건 + 상품 설명 30,000건 + 프로모션 기획안 20,000건 |
| 데이터 소스 | 자사 CMS 축적 데이터 + 공개 쇼핑몰 데이터 + 광고 카피 데이터셋 |
| 평가 메트릭 | BLEU, ROUGE-L, 도메인 전문가 Human Evaluation (5점 척도) |
| 학습 장비 | AWS g5.2xlarge (NVIDIA A10G 24GB) × 2대 |
| 예상 학습 시간 | 약 48시간 |
| 추론 최적화 | vLLM 서빙, 평균 응답 시간 2초 이내 목표 |

#### 핵심 기술 2: 비정형 디스플레이 Out-painting

**1) 기술 원리**
- **이론적 배경**: Diffusion Model 기반 이미지 확장 기술 (Rombach et al., "High-Resolution Image Synthesis with Latent Diffusion Models", CVPR 2022)
- **적용 원리**: 메인 오브젝트(상품 이미지)를 중심에 고정하고, 주변 여백을 Inpainting mask로 설정하여 문맥에 맞는 배경을 AI가 자동 생성. Latent space에서 조건부 생성을 수행하여 원본과 자연스러운 경계 구현
- **차별화**: 일반적인 Out-painting은 1:1 또는 16:9 비율 기준이나, 본 기술은 23.1인치(1920×158px, 약 12:1 비율)와 같은 극단적 가로 비율 디스플레이에 최적화

**2) 구현 절차**
```
[원본 이미지 입력] → [오브젝트 세그멘테이션(SAM)] → [중심 좌표 결정]
→ [타겟 해상도 설정(디바이스 정보 수신)] → [여백 마스크 생성]
→ [컨텍스트 프롬프트 생성(sLLM 연동)] → [Inpainting 실행(SD-XL)]
→ [경계 블렌딩(Gaussian blur + feathering)] → [Super-Resolution(Real-ESRGAN)]
→ [품질 검증(SSIM > 0.85)] → [CMS 포맷 패키징] → [디바이스 배포]
```

**3) 구체적 방법**
| 항목 | 상세 |
|------|------|
| 세그멘테이션 | Segment Anything Model (SAM, Meta) |
| 이미지 생성 | Stable Diffusion XL (SDXL 1.0) |
| 후처리 | Real-ESRGAN x4 (Super-Resolution) |
| 입력 해상도 | 512×512 (생성 기본) |
| 출력 해상도 | 1920×158 (23.1인치), 2560×158 (29인치), 커스텀 |
| 배경 생성 전략 | ControlNet (depth/canny) + 텍스트 프롬프트 |
| 품질 메트릭 | SSIM > 0.85, FID < 50, 육안 검수 통과율 99% |
| 처리 시간 | 이미지 1장당 5~8초 (GPU A10G 기준) |

#### 핵심 기술 3: Triple-Modal 입력 처리

**1) 기술 원리**
- Touch: 키오스크/사이니지의 터치 이벤트 로그를 시계열 데이터로 변환, LSTM/Transformer 기반 의도 분류
- Voice: Whisper Large V3 (OpenAI) 기반 STT + 리테일 용어 사전(Custom Vocabulary) 적용
- Text: 직접 텍스트 입력의 NLU(자연어 이해) 처리

**2) 구현 절차**
```
[멀티모달 입력 수신]
├── Touch → [좌표 로그 수집] → [세션 단위 그룹핑] → [행동 패턴 분류(CNN-LSTM)]
├── Voice → [오디오 캡처] → [노이즈 필터링(RNNoise)] → [STT(Whisper)] → [용어 보정]
└── Text  → [토큰화] → [의도 분류(BERT)]
→ [멀티모달 퓨전(Cross-Attention)] → [통합 의도 벡터 생성] → [sLLM 전달]
```

**3) 구체적 방법**
| 모달리티 | 모델 | 파라미터 | 목표 성능 |
|---------|------|---------|---------|
| Touch | CNN-LSTM (자체 개발) | 입력: 터치좌표+시간 시퀀스, 출력: 6개 의도 클래스 | 분류 정확도 95% |
| Voice | Whisper Large V3 + Fine-tuning | 리테일 용어 2,000개 + 다국어(한/영/일) | STT 인식률 98% |
| Text | BERT-base-multilingual | 의도 분류 10개 카테고리 | F1-score 0.92 |
| Fusion | Cross-Modal Attention | 3모달 임베딩 연결 | 통합 정확도 93% |

> 기술개요서에 미선정 사유 보완 내용을 최대한 압축하여 반영해야 함

#### 서면평가 배점별 작성 전략

| 평가지표 | 배점 | 핵심 작성 포인트 | 미선정사유 보완 |
|---------|------|---------------|-------------|
| **연구개발 지원 필요성** | 20점 | - 기업연구소 신설 3년 이내 → 초기 R&D 지원 절실<br>- 기존 H/W 기업 → AI 전환 필요성<br>- 경기도 소재 기업으로서 지역 기여 | - |
| **연구개발 계획의 적절성 및 목표달성가능성** | 40점 | - 핵심 기술 3가지의 **원리/절차/방법** 상세 기술 (T2)<br>- 성능지표별 **목표 설정 근거 + 측정 방법** 상세 (T3)<br>- **기존 기술 대비 비교분석** 표 포함 (T1) | T1, T2, T3 **핵심 보완** |

### 4.2 기술개요서 섹션별 작성 지침

#### 섹션 1: 과제 개요 (약 1페이지)
- 과제명: [디딤돌과 동일 또는 유사하게 유지]
- 연구 목표: 3가지 세부 목표 유지
- 연구 기간: 2026.6 ~ 2028.5 (2년)
- 총 연구비: 2억원 (지원금 1.5억 + 기관부담 5,000만원)

#### 섹션 2: 기술 현황 및 필요성 (약 2~3페이지) ← [T1] 보완 핵심
**반드시 포함할 내용:**
- [ ] 국내외 경쟁 기술 비교표 (6개 이상 경쟁사/기술)
- [ ] 각 기존 기술의 한계점 구체적 지적
- [ ] 자사 기술의 차별화 포인트 (정량적 수치)
- [ ] 기술 개발의 필요성 논리 전개

#### 섹션 3: 연구개발 내용 (약 3~5페이지) ← [T2] 보완 핵심
**각 핵심 기술마다 반드시 3단 구조 적용:**
- [ ] 기술 원리 (WHY): 이론적 배경, 학술 인용
- [ ] 구현 절차 (HOW): 플로우차트/단계별 프로세스
- [ ] 구체적 방법 (DETAIL): 모델/프레임워크/파라미터/데이터셋

**작성해야 할 핵심 기술 3가지:**
1. 리테일 특화 sLLM 파인튜닝 (LoRA/QLoRA)
2. 비정형 디스플레이 Out-painting (SDXL + SAM)
3. Triple-Modal 입력 처리 (Whisper + CNN-LSTM + BERT)

#### 섹션 4: 성능지표 및 달성 목표 (약 2~3페이지) ← [T3] 보완 핵심
**각 지표마다 반드시 포함:**
- [ ] 국내외 최고 수준 벤치마크 (출처 명시)
- [ ] 현재 수준 vs 목표 수준 GAP
- [ ] 목표 설정 근거 (기술적 접근법)
- [ ] 측정 환경 (HW/SW/네트워크 사양)
- [ ] 측정 절차 (사전조건, 반복횟수, 데이터셋)
- [ ] 결과치 계산 방법 (평균/중앙값/분산/CI)

#### 섹션 5: 사업화 계획 (약 2~3페이지) ← [B1~B4] 보완 핵심
- [ ] TAM/SAM/SOM 시장 규모 표 (출처 포함)
- [ ] 경쟁사 상세 비교표 (6개 이상)
- [ ] 타겟 고객 실명 리스트 + 접촉 현황
- [ ] 제품별 단가표 + Bottom-up 매출 산출
- [ ] SWOT 분석

#### 섹션 6: 연구 인력 및 인프라 (약 1~2페이지)
- [ ] 연구팀 7인의 학력/경력/역할
- [ ] 기업부설연구소 현황
- [ ] 보유 장비/인프라

## 5. 연구개발계획서 업그레이드 가이드 (2차 발표평가용)

### 5.1 디딤돌 계획서 대비 업그레이드 항목

| # | 디딤돌 원본 내용 | 문제점 | 업그레이드 방향 |
|---|-------------|-------|-------------|
| 1 | 경쟁사 일반 언급만 | 실명/정량 비교 없음 | **6개 경쟁사 상세 비교표** 추가 |
| 2 | 5-Step AI Workflow | 각 단계 HOW 부족 | **각 기술 3단 구조(원리/절차/방법)** 적용 |
| 3 | 성능지표 5개 + 간략 평가방법 | 근거/측정상세 부족 | **지표별 벤치마크+GAP+측정환경+절차+산출식** |
| 4 | 시장 "2,000억불" 1줄 | TAM/SAM/SOM 없음 | **3단계 시장 규모 + 출처** |
| 5 | 타겟 "대형유통사" | 실명 없음 | **6개 실명 기업 + 접촉현황** |
| 6 | 매출 "30억→80억→150억" | 산출식 없음 | **Bottom-up 산출 (단가×수량×채널)** |
| 7 | 경쟁사 3개 일반 언급 | 상세 분석 없음 | **6개 경쟁사 기술/사업 비교 + SWOT** |

### 5.2 추가해야 할 신규 콘텐츠

1. **기술 원리 다이어그램**: 각 핵심 기술별 아키텍처/플로우차트 3장 이상
2. **벤치마크 데이터 표**: 국내외 최고 수준 대비 자사 목표 비교 도표
3. **시장 세분화 도표**: TAM→SAM→SOM 깔때기 그래프
4. **경쟁사 포지셔닝 맵**: 2축(기술수준 vs 가격) 매트릭스
5. **Bottom-up 매출 산출표**: 연도별 × 채널별 × 제품별 3차원 분해
6. **LoRA 파인튜닝 상세 파라미터표**: 학습 설정 전체 공개
