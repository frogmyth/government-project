### 2.2 5-Step AI Workflow 상세 (원리/절차/방법)

#### **[Step 1] Triple-Modal 데이터 입력 및 수집**

| 모달리티 | 기술 원리 | 구현 방법 |
|----------|----------|----------|
| **Touch** | 정전용량 방식 터치 좌표 분석 | 클릭 좌표, 체류 시간, 스크롤 패턴을 Event Stream으로 수집 → Kafka 기반 실시간 처리 |
| **Voice** | 노이즈 캔슬링 + VAD(Voice Activity Detection) | RNNoise 전처리 → WebRTC VAD로 음성 구간 분리 → Whisper Large-v3 STT |
| **Text/Image** | NLP + Vision Transformer | 상품명/키워드 입력 시 BERT 임베딩 → 기존 이미지는 CLIP 인코딩으로 스타일 추출 |

**구현 절차:**
1. 디바이스에서 원시 데이터 수집 (1ms 단위)
2. Edge 서버에서 전처리 및 특징 추출 (10ms)
3. Cloud로 추론 요청 전송 (100ms)

#### **[Step 2] 리테일 특화 인식 및 전처리**

**Touch Log 분석 알고리즘:**
```
입력: touch_events = [(x, y, timestamp, duration), ...]
출력: behavior_label ∈ {가격비교중, 브랜드선호, 상세정보확인, ...}

1. 시퀀스 정규화: (x, y) → normalized_coords
2. 행동 패턴 분류: LSTM(normalized_coords, duration) → feature_vector
3. 레이블 예측: Softmax(MLP(feature_vector)) → behavior_label
```

**Retail STT 최적화 방법:**
- 유통 현장 은어 데이터셋 구축: "1+1", "매대", "엔드캡", "POP" 등 5,000개 용어
- Whisper Large-v3 + LoRA 파인튜닝 (학습 데이터 10만 발화)
- 다국어 지원: 한국어 기본 + 영어/일어 (H마트 진출 대비)

#### **[Step 3] LLM 기반 의도 파악 및 기획 (Reasoning)**

**리테일 sLLM 구축 방법:**

| 단계 | 방법 | 세부 내용 |
|------|------|----------|
| ① 데이터셋 | 유통 마케팅 데이터 10만건+ | 프로모션 문구, 계절별 카피, 상품 설명 (크롤링 + 파트너사 제공) |
| ② 베이스 모델 | Llama 3 8B | 오픈소스, 상업 라이선스 |
| ③ 파인튜닝 | LoRA (r=16, alpha=32) | 4-bit 양자화, 단일 A100 GPU로 학습 가능 |
| ④ 검증 | RAG 기반 할루시네이션 제어 | CMS 템플릿 DB 및 ERP/POS 데이터와 실시간 크로스체크 |

**RAG (검색 증강 생성) 파이프라인:**
```
Query → Embedding(BGE-M3) → Vector DB 검색(Milvus) → Top-K 문서 선택
      → sLLM 프롬프트 구성 → 응답 생성 → ERP 유효성 검증 → 최종 출력
```

#### **[Step 4] 프롬프트 엔지니어링 자동화**

**Context-Aware Prompting 알고리즘:**
```python
def generate_prompt(intent, context):
    # 의도별 스타일 파라미터 매핑
    style_params = {
        "고급스러움": {"lighting": "studio", "composition": "minimal", "color": "warm"},
        "긴급세일": {"lighting": "bright", "composition": "dynamic", "color": "red_accent"},
        "계절프로모션": {"lighting": "natural", "composition": "seasonal", "color": "context"},
    }
    
    base_prompt = f"Create a marketing image for {context.product_name}"
    style = style_params.get(intent, default_style)
    
    # Negative Prompt 자동 삽입 (품질 제어)
    negative = "text artifacts, distorted objects, watermark, low quality"
    
    return f"{base_prompt}, {style}, --neg {negative}"
```

#### **[Step 5] 콘텐츠 생성 및 배포 최적화**

**Out-painting 파이프라인 (비정형 디스플레이 대응):**

```
입력: 1024x1024 원본 이미지, 목표 해상도 3840x158 (Bar-Type)

1. 중심 오브젝트 감지: SAM(Segment Anything Model)로 상품 영역 추출
2. 배경 확장: Stable Diffusion Inpainting으로 좌우 여백 생성
3. Super-Resolution: Real-ESRGAN으로 4K 업스케일링
4. 최종 크롭: 목표 해상도에 맞춰 중앙 정렬 + 트리밍

출력: 3840x158 최종 이미지 (< 3MB, WebP 포맷)
```

**CMS 자동 연동:**
- 생성된 콘텐츠를 AIsirius CMS 규격(JSON 메타데이터 + 미디어 파일)으로 패키징
- MQTT 프로토콜로 매장 내 디바이스에 실시간 배포 (지연시간 < 2초)
