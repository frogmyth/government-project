| **3D 시뮬레이션** | ✅ 운영 | Three.js 기반 매장 3D 렌더링 |

**AI 제안 엔진 (aiSuggestionEngine.ts):**
```
📊 3가지 AI 최적화 컨셉:

1. 💰 매출 극대화 (revenue-max)
   - 매출 상위 20% → 골든존(눈높이) 집중 배치
   - 상위 상품 페이싱 확대 (품절 방지)
   - 하위 20% 하단 이동 → 공간 확보
   → 예상 매출 10~18% 증가

2. 📈 마진 최적화 (margin-opt)
   - Q1(고매출+고마진) 골든존 집중
   - Q3(저매출+고마진) 눈높이 배치로 가시성 향상
   - Q4(저매출+저마진) 축소/교체
   → 예상 마진 10~20% 개선

3. 🛒 동선 최적화 (flow-opt)
   - 충동구매 상품(과자, 빵) → 입구/눈높이
   - 보완 카테고리 인접 배치 (라면↔조미료, 빵↔유제품)
   - 필수품(물) 안쪽 배치 → 동선 확장
   → 교차 판매 8~13% 증가
```

**자동 배치 알고리즘 (autoPlacement.ts):**
```python
# 골든존 최적화 규칙
1. 골든존(위에서 2단) = 매출 상위 20%
2. 눈높이 아래(3단) = 매출 상위 20~50%
3. 상단(1단) = 경량 + 고마진 상품
4. 하단(4단) = 대용량/저회전 상품
5. 같은 카테고리는 인접 배치 유지
6. 비슷한 높이끼리 같은 선반 배치

# 페이싱 자동계산 공식
facing = ceil(상품 매출 비중 × 선반 총 슬롯 수)
- 최소 1, 최대 50% (한 상품이 선반의 절반 초과 방지)
- 골든존 상위 상품 +1 보너스 페이싱
- 빈 공간은 매출 높은 순으로 페이싱 확장
```

#### **📋 [고도화] AI 작업 단위 자동 생성**

**기존 기능**: 수동 작업 단위 생성 (작업명, 상품, 기간 설정)

**고도화 목표**:
- **패턴 학습**: 과거 작업 단위 분석 → 다음 작업 자동 제안
- **시즌 캠페인**: 계절/공휴일 기반 프로모션 자동 생성
- **재고 연동**: 재고 부족 시 대체 상품 자동 교체
- **성과 분석**: 작업 종료 후 AI 리포트 자동 생성
