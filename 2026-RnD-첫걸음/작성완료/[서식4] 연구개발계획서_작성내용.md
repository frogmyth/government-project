## 연구개발 방법 (T2 보완: 원리/절차/방법 3단 구조)

### [핵심기술 1] 리테일 특화 sLLM 파인튜닝 (AI 챗봇 CMS 핵심)

**1) 기술 원리 (WHY & WHAT)**
- 이론적 배경: 대규모 언어모델(LLM)의 도메인 적응을 위한 LoRA (Low-Rank Adaptation, Hu et al., 2021, arXiv:2106.09685)
- 적용 원리: 사전학습 가중치 행렬 W에 저차원 분해 ΔW = BA를 추가, 전체 파라미터의 0.1~1%만 학습하면서 풀 파인튜닝 근접 성능 달성
- 수식: h = Wx + ΔWx = Wx + BAx (B ∈ R^{d×r}, A ∈ R^{r×k}, r << min(d,k))

**2) 구현 절차 (HOW - Process)**
```
[데이터 수집] → [전처리/클리닝] → [Instruction-Response 포맷팅]
→ [베이스 모델 로딩(Llama 3 8B)] → [LoRA 어댑터 설정(rank=16, alpha=32)]
→ [4-bit QLoRA 학습 실행] → [Validation Loss 모니터링]
→ [모델 병합(Merge)] → [추론 최적화(vLLM/TensorRT)] → [FastAPI 서빙]
→ [RAG 파이프라인 구축] → [Tool Calling 개발] → [챗봇 CMS v1.0]
```

**3) 구체적 방법 (DETAIL)**

| 항목 | 상세 |
|------|------|
| 베이스 모델 | Meta Llama 3 8B-Instruct (HuggingFace) |
| 파인튜닝 기법 | QLoRA (4-bit quantization + LoRA) |
| LoRA 파라미터 | rank=16, alpha=32, dropout=0.05 |
| 학습률 | 2e-5 (cosine scheduler, warmup 10%) |
| 에폭 | 5 epochs |
| 배치 사이즈 | 4 (gradient accumulation 4 = effective 16) |
| 학습 데이터 | 리테일 마케팅 문구 50,000건 + 상품 설명 30,000건 + 프로모션 기획안 20,000건 |
| 데이터 소스 | 자사 CMS 축적 데이터 + 공개 쇼핑몰 데이터 + 광고 카피 데이터셋 |
| 평가 메트릭 | BLEU, ROUGE-L, Human Evaluation (5점 척도) |
| 학습 장비 | AWS g5.2xlarge (NVIDIA A10G 24GB) × 2대 |
| 예상 학습 시간 | 약 48시간 |
| 추론 최적화 | vLLM 서빙, 평균 응답 시간 2초 이내 목표 |

### [핵심기술 2] 비정형 디스플레이 Out-painting (생성형 AI 고도화 핵심)

**1) 기술 원리 (WHY & WHAT)**
- 이론적 배경: Diffusion Model 기반 이미지 확장 기술 (Rombach et al., "High-Resolution Image Synthesis with Latent Diffusion Models", CVPR 2022)
- 적용 원리: 메인 오브젝트(상품)를 중심 고정, 주변 여백을 Inpainting mask로 설정하여 문맥에 맞는 배경 AI 자동 생성
- 차별화: 일반 Out-painting은 1:1 또는 16:9 비율이나, 본 기술은 23.1인치(1920×158px, 약 12:1 비율) 극단적 가로 비율에 최적화

**2) 구현 절차 (HOW - Process)**
```
[원본 이미지 입력] → [오브젝트 세그멘테이션(SAM)] → [중심 좌표 결정]
→ [타겟 해상도 설정(디바이스 정보 수신)] → [여백 마스크 생성]
→ [컨텍스트 프롬프트 생성(sLLM 연동)] → [Inpainting 실행(SDXL)]
→ [경계 블렌딩(Gaussian blur + feathering)] → [Super-Resolution(Real-ESRGAN)]
→ [품질 검증(SSIM > 0.85)] → [CMS 포맷 패키징] → [디바이스 배포]
```

**3) 구체적 방법 (DETAIL)**

| 항목 | 상세 |
|------|------|
| 세그멘테이션 | Segment Anything Model (SAM, Meta) |
| 이미지 생성 | Stable Diffusion XL (SDXL 1.0) |
| 후처리 | Real-ESRGAN x4 (Super-Resolution) |
| 입력 해상도 | 512×512 (생성 기본) |
| 출력 해상도 | 1920×158 (23.1인치), 2560×158 (29인치), 커스텀 |
| 배경 생성 전략 | ControlNet (depth/canny) + 텍스트 프롬프트 |
| 품질 메트릭 | SSIM > 0.85, FID < 50, 육안 검수 통과율 99% |
| 처리 시간 | 이미지 1장당 5~8초 (GPU A10G 기준) |

### [핵심기술 3] Triple-Modal 입력 처리 (음성/터치/텍스트 통합)

**1) 기술 원리 (WHY & WHAT)**
- Touch: 터치 이벤트 시계열 데이터를 CNN-LSTM 하이브리드로 의도 분류
- Voice: Whisper Large V3 (OpenAI) 기반 STT + 리테일 Custom Vocabulary 적용
- Text: 직접 텍스트 입력의 NLU(자연어 이해) 처리 (BERT 기반)
- 선행연구: Li et al., "Understanding User Intent from Touch Interactions", CHI 2023; Radford et al., Whisper, 2023

**2) 구현 절차 (HOW - Process)**
```
[멀티모달 입력 수신]
├── Touch → [좌표 로그 수집] → [세션 그룹핑] → [행동 패턴 분류(CNN-LSTM)]
├── Voice → [오디오 캡처] → [노이즈 필터링(RNNoise)] → [STT(Whisper)] → [용어 보정]
└── Text  → [토큰화] → [의도 분류(BERT)]
→ [멀티모달 퓨전(Cross-Attention)] → [통합 의도 벡터] → [sLLM 전달]
```

**3) 구체적 방법 (DETAIL)**

| 모달리티 | 모델 | 핵심 파라미터 | 목표 성능 |
|---------|------|-------------|---------|
| Touch | CNN-LSTM (자체 개발) | 입력: 터치좌표+시간, 출력: 6개 의도 클래스 | 분류 정확도 95% |
| Voice | Whisper Large V3 + Fine-tuning | 리테일 용어 2,000개 + 다국어(한/영/일) | STT 인식률 98% (WER 2%) |
| Text | BERT-base-multilingual | 의도 분류 10개 카테고리 | F1-score 0.92 |
| Fusion | Cross-Modal Attention | 3모달 임베딩 연결 | 통합 정확도 93% |

### [핵심기술 4] AI 롤링 최적화 + 플래노그램 고도화

**1) 기술 원리**
- 롤링 최적화: Multi-armed Bandit (Thompson Sampling) 기반 콘텐츠 노출 순서 실시간 최적화
- 플래노그램: YOLO v8 기반 상품 인식 + 기상청 API 연동 수요 예측

**2) 구현 절차**
```
[롤링] 조회수 수집 → 보상 함수 계산 → Thompson Sampling → 최적 순서 결정 → A/B 테스트
[플래노그램] 카메라 촬영 → YOLO 상품 인식 → 준수율 계산 → 날씨 API → 수요 예측 → 자동 재배치
```

**3) 구체적 방법**

| 기술 | 모델/방법 | 핵심 파라미터 | 목표 성능 |
|------|----------|-------------|---------|
| 롤링 | Thompson Sampling | Beta(α, β) 분포, 탐색-활용 비율 자동 조정 | CTR 30% 향상 |
| 상품 인식 | YOLOv8-L | mAP@0.5 > 0.95, 100개 SKU 학습 | 준수율 측정 95% |
| 수요 예측 | LSTM + 기상청 API | 3일 선행 예측, 온도/강수/습도 입력 | 정확도 85% |

- AI 기반 마케팅 콘텐츠 자동 생성 기능 전무

◦ **리테일 미디어 네트워크(RMN) 성장**
- 2025년 600억 달러 → 2028년 1,500억 달러 예상
- 매장 내 디지털 마케팅의 중요성 급증

### 1-1-2. 기술적 필요성

◦ **AI 콘텐츠 자동 생성 필요**
- 중소 리테일: 전문 디자이너 부재, 마케팅 콘텐츠 제작 어려움
- 대형 리테일: 수천 개 상품의 콘텐츠 제작 시간/비용 과다

◦ **비전문가 친화적 인터페이스 필요**
- 기존 CMS: 복잡한 메뉴 구조, 학습 곡선 높음
- 매장 직원: IT 비전문가, 직관적 조작 필요

◦ **AI 기반 최적화 필요**
- 콘텐츠 롤링: 수동 설정 → AI 자동 최적화
- 플래노그램: 경험 기반 → 데이터 기반 배치

### 1-1-3. 경제적·사회적 필요성

◦ **ESG 경영 강화**
